# Класове и обекти. Конструктори и деструктори. Използване на член-функции

## Член-функции
**Член-функциите** са вътрешни функции за клас/структура, които работят с техните член-данните

```c++
struct Time {
	int hour;
	int minute;
	int second;
};

bool isMorning(const Time& obj) {
	return obj.hour >= 0 && obj.hour < 12;
}
```
Това може да се пренапише като:
```c++
struct Time {
	int hour;
	int minute;
	int second;
	
	bool isMorning() const {
		return hour >= 0 && hour < 12;
	}
};
```

Тук може да забележим две неща:
* не ни се налага да подаваме обект на член-функцията - тя сама "върви в комплект" с указател `this`, който сочи към конкретния обект. Компилатора сам го добавя към всяка една член-фунцкия за всеки един клас/структура, *така че не ни се налага да го пишем.*
Особено важно е в  случаи, когато имаме повторение на член-данна и параметър на функция:
```c++
struct A {
	int value;
	
	void setValue(int value) { // за компилатора това е все едно void setValue(A* const this, int value)
		// value = value -> грешка: няма как да знаем за кое value става въпрос
		this->value = value; // така вече оказваме, че искаме да дадем стойност на член-данната value (this->value), която е равна на параметъра на функцията със същото име
	}
};
```
* в края на `bool isMorning()` може да видите, че се появи думата `const`. С нея оказваме, че вътрешното състояние на класа/структурата (т.е. член-данните) няма да бъдат променяни. Това се прави от една страна за по-добра четимост и работа от страна на програмистите (**const-correctness**), от друга за оптимизация за компилатора.
	* *неконстантни* член-функции могат да променят състоянието на класа, така че могат да викат както константни, така и неконстанти фунцкии
	* *константните* член-функции обаче могат да викат в себе си само други константни член-функции
```c++
class Container {
	int* arr;
	int size;
public:
	Container(int s) : size(s) {
		arr = new int[size];
	}

	void printSize() const { // може да се третира като void printSize(const Container* const this)
		std::cout << "Size: " << size << std::endl;
		size = 0; // тук ще ни даде грешка -> опитваме се да променим size в константна член-фунцкия
	}
};

int main() {
	Container c(10);
	c.printSize();
	return 0;
}
```

## Конструктори и деструктори
Конструкторите и деструкторите са специален вид член-функции, които се използват съответно за създаване и изтриване на обектите.

**Жизнен цикъл на обект**:

-   Създаване на обект в даден scope – заделя се памет и член-данните се инициализират.
-   Достига се до края на scope-a.
-   Обектът и паметта, която е асоциирана с него, се затрива.

### Конструктор
-   Извиква се веднъж - при създаването на обекта
-  Има същото име като класа/ структурата и няма тип на връщане (връща констатна референция към новосъздадения обект)
-  Може да се overload-ва -> т.е. можем да имаме няколко конструктора с различни параметри, но за създаването на конкретен обект се извиква само един от всичките контруктори, точно веднъж
-   Задава стойности на член-данните на класа/ структурата (в тялото си или чрез  **member initializer list**)

#### Конструктор без параметри - конструктор по подразбиране (default constructor)
Ако ние не сме разписали никакви констуктори, компилатора сам си генерира default-ен конструктор
```c++
struct A
{
    
};

/* същото като това:
struct A
{
    A() {};
};
*/
```
Но ако сме разписали някои контруктор:
```c++
class B {
	int b; 
	public:
		B(int b) {
			this->b = b;
		}
	// в този случай вече нямаме B(), ако искаме да го
	// използваме трябва да го разпишем ние сами
}
``` 
А защо ни е да имаме конструктор по подразбиране? Когато имаме масиви и/или указатели, компилатора извиква този конструктор:
```c++
struct A
{
    
};

class B {
	int b; 
	public:
		B(int b) {
			this->b = b;
		}
};

int main()
{
    A arr[10]; // ще задели масив за 10 обекта от тип A, като ще извика конструктора им по подразбиране
    A* ptr = new A(); // отново, ще извика default-ния конструктор
    B array[5] // но това няма да се компилира -> нямаме конструктор по подразбиране
}
```

### Деструктор
-   Извиква се веднъж - при изтриването на обекта.
-   Има същото име като класа със символа '~' в началото.
-   За разлика от конструкотрите, може да има само един деструктор - **не може да се overload-ва**
- Няма указан тип на връщане

## Конструктори и деструктор при влагане на обекти
```c++
class A {
public:
	A() {
		std::cout << "A()" << std::endl;
	}

	~A() {
		std::cout << "~A()" << std::endl;
	}
};

class B {
	A a;
public:
	B() {
		std::cout << "B()" << std::endl;
	}
	~B() {
		std::cout << "~B()" << std::endl;
	}
};

class C {
	A a;
	B b;
public:
	C() : b(), a() { // тук редът на инициализиращия списък не е от значение - първо ще се създаде A, след което B
		std::cout << "C()" << std::endl;
	}
	~C() {
		std::cout << "~C()" << std::endl;
	}
};

int main() {
	C* ptr = new C();
	return 0;
}
```
В дадения пример имаме клас `C`, с композирани в него класове `A` и `B`, който от своя страна има още една инстанция на `A` в себе си. Можете да видите как е представен C  в паметта (това е общ пример, предполагаме че класовете ни ще са по-съществени):

![](/Week03/resources/c.png)

Както може да видите, на първо място седи `A` и после `B`, тъй като така сме ги подредили в класа си - тази подредба е много важна, тъй като това е и реда на инициализация на обектите, независимо от реда в инициализиращия списък.

Съответно, когато извикаме конструктора на `C`, първо се създава `A`, след което `B`, което от своя страна вътрешно инициализира `A`, като най-накрая вече се създава цялостното `C`. (накрая на конзолата ще бъде отпечатано A(), A(), B(), C())

Може да видите по-добре какво се случва в стека на картинката:

![](/Week03/resources/stack.png)

Накрая, когато извикаме деструктора, трябва да премахнем нещото на върха на стека - `C`. Затова и редът на деструкторите е обратен на реда на конструкторите. (след изтриването на обекта на конзолата ще се изпише ~C(), ~B(), ~A(), ~A())
