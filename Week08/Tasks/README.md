# Задачи

## 1. Vector
Да се реализира шаблонен клас `Vector<T>`, който представлява динамичен масив от обекти. Той има следните методи:
* **Vector()** - конструктор по подразбиране; *size* = 0, *capacity* = 8 (или друга стойност по подразбиране)
* **Vector(size_t n)** - size = 0, capacity = n
* **Vector(size_t n, const T& obj)** - size = capacity = n, всеки елемент в масива е равен на *element*
* **size()** - връща *size* - текущия ни брой елементи
* **capacity()** - връща *capacity* - колко е максималната вместимост на масива *към момента* (това е динамичен масив, така че когато *size == capacity* учвеличаваме капацитета)
* **push_back(const T& obj)** - добавя елемента в края на масива
* **push_back(T&& obj)**
* **insert(size_t idx, const T& obj)** - добавя елемента на подадения индекс. В случай, че индекса е невалиден метода не прави нищо
* **insert(size_t idx, T&& obj)**
* **erase(size_t idx)** - премахва елемента на подадения индекс. Ако той е невалиден, метода не прави нищо
* **begin()** - връща указател към началото на масива (*реализирайте с константен и неконстантен достъп*)
* **end()** - връща указател към края на масива (*реализирайте с константен и неконстантен достъп*)
* **operator[](size_t idx)** - връща елемента на подадения индекс (*реализирайте с константен и неконстантен достъп*)
* **shrink_to_fit()** - намалява капацитета до текущия размер и преоразмерява масива
* **empty()** - връща дали масива е празен
* **clear()** - премахва всички елементи и задава size = 0

## 2. Pair
Реализирайте структура `Pair<K, V>`, която представялва наредена двойка *<ключ, стойност>*. Има следните член-данни:
* **first** - първи елемент, ключа
* **second** - втори елемент, стойност

Реализирайте следните методи:
* **operator ==**
* **operator !=**

## 3. Map
Реализирайте клас `Map<K, V>`, който представлява съвкупоност от наредени двойки *<ключ, стойност>*. Тук ключовото е, че на един ключ може да съответства само една стойност, т.е. нямаме повторение на ключове. Има следните методи:
* **add(const K& key, const V& value)** - добавя нова наредена двойка, ако ключа не присъства в map-a
* **add(const K& key, V&& value)**
* **remove(const K& key)** - премахва наредената двойка, съответсваща на дадения ключ (ако съществува, в противен случай метода не прави нищо)
* **contains(const K& key)** - връща дали дадения ключ съществува в map-a
* **operator[](const K& key)** - връща ни стойността, съответстваща на подадения ключ
* **size()**
* **capacity()**
* **empty()**

## 4. Stack
Реализирайте шаблонен клас `Stack<T>`, който представлява структура данни, базирана на принципа **LIFO - Last In, First Out** (т.е. последния елемент, който сме вкарали, е първия елемент, който можем да достъпим в стека). 

Представете си го като купчина карти на масата - трябва първо да вземем най-горната (т.е. последната) карта, за да разгледаме купчината.

Има следните методи:
* **push(const T& obj)** - вкарва елемент в стека на последно място
* **push(T&& obj)**
* **pop()** - премахва елемента на последно място в стека, **без да го връща** (т.е. е void)
* **top()** - връща ни последния ("най-горния") елемент от стека. **Има единствено константен достъп**
* **empty()**
* **size()**

## 5. Queue
Реализирайте шаблонен клас `Queue<T>`, който представлява структура данни, базирана на принципа **FIFO - First In, First Out** (т.е. първият елемент, който сме вкарали, е първия елемент, който можем да достъпим в опашката). 

Представете си го като опашка в магазина - първият, наредил се на опашката, е първият обслужен

Има следните методи:
* **push(const T& obj)** - вкарва елемент в опашката на последно място
* **push(T&& obj)**
* **pop()** - премахва елемента на *първо* място в опашката, **без да го връща** (т.е. е void)
* **front()** - връща ни първия ("най-предния") елемент от опашката. **Има единствено константен достъп**
* **back()** - връща ни първия ("най-предния") елемент от опашката. **Има единствено константен достъп**
* **empty()**
* **size()**
