# Разделна Компилация

Една програма на C++ може да бъде разбита на множество изходни файлове (.cpp), които се компилират независимо един от друг. **Така промяната на един файл не води до прекомпилиране на всички останали.**

Основни стъпки при изграждане на изпълнима програма:
* препроцесор - обработка/изпълнение на всички (препроцесорни) директиви (нещата, запозващи с #)

  * #include <име-на-файл> - този ред се заменя със съдържанието на въпросния файл (това не е видимо за нас, а за самата програма). Така към всеки файл с изходен код (.cpp, .c, ...) се добавят всички редове код от други файлове (.h, .hpp, ...) и се генерира пълния изходен код - още известен като предварително обработен изходен файл.

  * #pragma - директива, която казва как да се компилира даден код
    * pragma once - конкретния файл трябва да се обработи веднъж
  
  * #define - дефинира macro - именувана част от кода
  ```c++
  #define text1 text2 // всяко срещане на text1 се заменя с text2
  #define МАX 10 // друг начин за дефиниране на константи
  
    /* може да се третира и като малки фунцкии. 
     * Тук проблема е, че всичко се заменя буквално и няма обработка на подадените параметри 
     * (ако подадем a + 1, няма да пресметнем на колко е равно, а буквално ще се замени с израза a + 1)*/
  #define square(x) x * x
  ```

* компилация - всеки предварително обработен изходен файл се компилира в обектен файл (.o, .obj)
  * .cpp -> .o/.obj


* linking - всички обектни файлове се свързват в едно, за да се получи крайният изпълним файл/изпълнима програма (.exe)

Оттук нататък ще следваме правилото:

**В .h файлове се пишат само декларации**

**В .cpp файлове разписваме дефинициите**

# Static

### Променливи
* static <тип> <име> = <стойност>
* пазят се в scope-a/паметта на глобалните променливи - въпреки това те не са задължително глобални, освен ако не ги дефинираме като такива
* променя продължителността на променливата от automatic (до края на съответния scope) на static (до края на програмата)
* **задължително се задава стойност при инициализацията им**

```c++
// това е създадено само за примера, не правене степени на 2ката по този начин :)
int powerOfTwo()
{
    static int result = 1;
    result *= 2;
    return result;
}

int main()
{
    /*въпреки че привидно създаваме нов result при всяко извикване на powerOfTwo, 
     *понеже е static, стойността му се запазва, т.е. сме създали променливата само веднъж и 
     *заради това е важно да се даде стойност при инициализацията им*/
    std::cout << powerOfTwo() << std::endl; //2
    std::cout << powerOfTwo() << std::endl; //4
    std::cout << powerOfTwo() << std::endl; //8
}
```

### Функции
* обръзват се с конкретния .cpp файл, в който са, и не могат да се използват извън него

### Член-данни на клас
* обръзват се с целия клас, а не конкретен обект от него
* съответно се споделя от всички инстанции на класа
* инициализира се извън класа

### Член-фунцкии
* могат да работят само със static член-данни (но обратното не е вярно)
* нямат указател `this`, тъй като са общи за целия клас, а `this` сочи към текущия обект, т.е. конкретна инстанция на класа

```c++
class A
{   
public:
    static int counter;
    
    /* не-static член-фунцкия може да достъпи static член-данна, дори с подаден параметър*/
    void multiply(int a)
    {
        counter *= a;
    }
    
    static void print()
    {
        //... някаква логика ...
    }
};

int A::counter = 1; // инициализация извън класа -> чрез оператора за резолюция ::

int main()
{
    A a1, a2;
    a1.multiply(3);
    a2.multiply(2);
    std::cout << A::counter << std::endl; // 6
    A::print() // не ни трябва обект, директно викаме фунцкията чрез класа
}

```
